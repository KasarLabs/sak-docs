# Agent Capabilities

The Starknet Agent Kit provides a powerful set of capabilities for interacting with the Starknet blockchain using natural language processing.

## Core Capabilities

### Natural Language Processing

The agent can understand and execute commands written in natural language:

```typescript
// These commands all work:
await agent.execute('Check my ETH balance');
await agent.execute('What is my USDC balance?');
await agent.execute('Send 1 ETH to vitalik.eth');
```

### Smart Contract Interactions

The agent can interact with smart contracts:

```typescript
// Deploy contracts
await agent.execute('Deploy my NFT contract');

// Read contract data
await agent.execute('Get total supply of token 0x123...');

// Write to contracts
await agent.execute('Mint 1 NFT from collection 0x456...');
```

### DeFi Operations

Built-in support for common DeFi operations:

```typescript
// Swaps
await agent.execute('Swap 100 USDC for ETH');

// Liquidity provision
await agent.execute('Add liquidity to ETH/USDC pool');

// Yield farming
await agent.execute('Stake 1000 USDC in Aave');
```

## Advanced Features

### Transaction Management

The agent includes built-in transaction monitoring:

```typescript
const txHash = await agent.execute('Send 1 ETH to 0x123...');
await agent.monitorTransaction(txHash);
```

### Error Handling

Comprehensive error handling for all operations:

```typescript
try {
  await agent.execute('Swap 1000 ETH for USDC');
} catch (error) {
  if (error instanceof AgentExecutionError) {
    console.error('Execution failed:', error.message);
  } else if (error instanceof StarknetTransactionError) {
    console.error('Transaction failed:', error.message);
  }
}
```

### Custom Tool Integration

You can extend the agent's capabilities with custom tools:

```typescript
StarknetToolRegistry.registerTool({
  name: 'custom_tool',
  description: 'Custom tool description',
  schema: yourZodSchema,
  execute: async (agent, params) => {
    // Your implementation
  }
});
```

## Best Practices

1. Always validate inputs before execution
2. Implement proper error handling
3. Monitor transaction status
4. Use batch transactions when possible
5. Implement rate limiting for AI operations
